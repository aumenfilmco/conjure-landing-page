    **Updated FeatureRow:**
    ```tsx
    function FeatureRow({ feature, index, activeIndex, onActivate }: FeatureRowProps) {
      const { ref } = useInView({
        rootMargin: '-40% 0px -40% 0px',
        threshold: 0,
        onChange: (inView) => {
          if (inView) onActivate(index)
        },
      })

      const isActive = activeIndex === index

      return (
        <div
          ref={ref}
          data-testid="feature-row"
          className={isActive ? 'text-primary' : 'text-muted-foreground'}
        >
          <h3 className={`font-sans font-medium text-xl mb-3 ${isActive ? 'text-foreground' : ''}`}>
            {feature.TITLE}
          </h3>
          <p className="text-sm leading-relaxed">{feature.OUTCOME}</p>
        </div>
      )
    }
    ```

    **Why 6 IntersectionObserver calls:** `react-intersection-observer` creates one native IntersectionObserver instance per `useInView` hook instance (with the same `rootMargin` options). Since there are 6 `FeatureRow` components each calling `useInView`, the mock constructor will be called 6 times total when the component mounts. The FLYT-03 test asserts exactly this.

    No other changes to the file. The crossfade image stack from Task 1 already drives `activeIndex` changes correctly — `onActivate(index)` calls `setActiveIndex(index)` which triggers a re-render with the new active image getting `opacity-100`.
