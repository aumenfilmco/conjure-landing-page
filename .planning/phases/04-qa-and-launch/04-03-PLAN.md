**Step A — Compress hero-screenshot.webp using sharp (available globally at Node level).**

Run a one-off Node script from the repo root to re-compress the image at quality 70 (lossy WebP). The current file is 73KB — compressing to q70 typically yields 40-60KB for this type of screenshot content without visible degradation.

Run from `/Users/christopheraumen/Documents/Conjure-Landing-Page` (the repo root, not the Next.js subdirectory):

```bash
node -e "
const sharp = require('sharp');
const path = require('path');
const input = path.resolve('conjure-landing-page/public/hero-screenshot.webp');
sharp(input)
  .webp({ quality: 70, effort: 6 })
  .toFile(input + '.tmp')
  .then(info => {
    require('fs').renameSync(input + '.tmp', input);
    console.log('Compressed:', info);
  })

"
```

**Step B — Add `sizes` prop to the next/image element in HeroSection.tsx.**

Open `conjure-landing-page/src/components/sections/HeroSection.tsx`. Locate the `<Image>` element for the hero screenshot (around line 160–167). Add a `sizes` prop between `priority` and `className`:

Current state of the Image element:
```tsx
<Image
  src="/hero-screenshot.webp"
  alt="Conjure storyboard deck — beach proposal, golden hour, branded Google Slides export"
  width={2984}
  height={1865}
  priority
  className="w-full h-auto"
/>
```

Updated (add `sizes` prop between `priority` and `className`):
```tsx
<Image
  src="/hero-screenshot.webp"
  alt="Conjure storyboard deck — beach proposal, golden hour, branded Google Slides export"
  width={2984}
  height={1865}
  priority
  sizes="(max-width: 1024px) 100vw, 60vw"
  className="w-full h-auto"
/>
```

Rationale: On mobile (max-width 1024px) the image spans full viewport width; on desktop it fills roughly 60% of the viewport (the left column in the hero layout). Without this hint, Next.js defaults to `100vw` at all breakpoints, causing it to include the full 2984px source dimension in its srcset and potentially serving an oversized image to mobile clients.
