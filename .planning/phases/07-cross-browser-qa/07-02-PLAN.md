Glass surface CSS (globals.css):
- Fallback (outside @supports): solid background oklch(0.1x 0 0 / 0.9x)
- Enhancement (inside @supports): -webkit-backdrop-filter: blur(18px); backdrop-filter: blur(16px) saturate(180%)

- Mint orb gradient: oklch(0.92 0.18 142 / 0.44) at card corner — this is the region most likely to reduce contrast

FeaturesSection sticky panel:
- Desktop: right column has position:sticky, top: 8rem (inline style)
- Prerequisite: no overflow:hidden on section or grid container (set in Phase 5/6)
- Prerequisite: no transform ancestor — FadeInWrapper removed in Phase 5
- Mobile: md:hidden hides desktop grid; features-mobile-stack shows stacked cards

Scroll-spy:
- rootMargin: '-40% 0px -40% 0px' — only the center 20% of viewport fires
- Fast scroll: IntersectionObserver batches callbacks — last row to intersect before scroll stops is what fires
- Expected behavior: correct row highlights within one scroll-stop after fast scroll
</interfaces>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 5: WCAG AA contrast on .glass-surface text</name>
  <what-built>Glass cards use near-white foreground text (oklch(0.98 0 0), approximately #F9F9F9) over a frosted dark background. The background composites a dark glass layer (oklch(0.14 0 0 / 0.45)) over near-black (oklch(0.04 0 0)). The effective contrast is very high — but the mint green orb gradient (oklch(0.92 0.18 142 / 0.44)) at the card corner lightens the background in that region and may reduce contrast locally.</what-built>
  <how-to-verify>
    Open http://localhost:3000 in Chrome. Navigate to the Pricing section (glass tier cards are the primary glass-surface elements with body text).

    **Method 1 — Chrome DevTools color picker (recommended):**
    1. Open DevTools (F12) → Elements panel
    2. Select a Pricing tier card text element (e.g., the price label "$59/mo" or the feature list items)
    3. In the Styles panel, click the text color swatch (near-white) — the color picker opens
    4. Scroll to the "Contrast ratio" section at the bottom of the picker — Chrome reports the ratio automatically
    5. Confirm ratio ≥ 4.5:1 (AA threshold for normal text)

    **Check the orb corner specifically:**
    6. Scroll to the corner of a card where the mint green orb gradient is brightest (top-left or top-right of a card)
    7. If text appears in that region, check its contrast there too
    8. If no body text sits in the orb corner, note that the corner is decoration-only

    **Method 2 — WebAIM (if DevTools ratio not visible):**
    - Screenshot the rendered glass card
    - Go to https://webaim.org/resources/contrastchecker
    - Use the eyedropper to sample foreground (text) and background (the glass area behind text)

    Confirm:
    - [ ] Body text contrast ratio ≥ 4.5:1 at card center
    - [ ] Body text contrast ratio ≥ 4.5:1 near the orb corner (or confirm no body text sits in the brightest orb region)
    - [ ] Heading text contrast ratio ≥ 4.5:1 (if headings are larger, AA large = 3:1, but 4.5:1 is safer)
  </how-to-verify>
  <resume-signal>Type "contrast: pass" with the measured ratio (e.g. "contrast: pass — 14.5:1 body text, 8.2:1 near orb corner"), or describe any failures with the measured ratio and which element failed.</resume-signal>
